# TCC × MacroDroid Integration Spec
## Addendum to TCC_Replit_Build_Spec.md · v1.0

---

# WHAT THIS DOES

Tony's Android phone becomes the telephony layer for TCC. MacroDroid (installed on Tony's phone) bridges the native dialer/SMS app to the Replit web app via webhooks.

| Action | How it works |
|--------|-------------|
| Tony taps "Call" in TCC | Browser opens `tel:` link → Android dialer opens → Tony taps green button |
| Call ends | MacroDroid detects it → POSTs number + duration + timestamp to TCC API |
| Tony sends SMS from TCC | TCC API → MacroDroid webhook trigger → phone sends SMS natively |
| Someone texts Tony back | MacroDroid detects incoming SMS → POSTs sender + body + timestamp to TCC API |
| Someone calls Tony | MacroDroid detects incoming call → POSTs number + timestamp to TCC API |

# STEP 1: ADD SUPABASE TABLE

Run this SQL (add to existing schema):

```sql
-- Phone activity log (calls + SMS, inbound + outbound)
CREATE TABLE phone_log (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  contact_id UUID REFERENCES contacts(id),
  contact_name TEXT,
  phone_number TEXT NOT NULL,
  type TEXT NOT NULL,          -- 'call_outbound', 'call_inbound', 'sms_outbound', 'sms_inbound'
  duration_seconds INT,        -- calls only
  sms_body TEXT,               -- SMS only
  logged_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  matched BOOLEAN DEFAULT FALSE, -- did system match to a contact?
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Index for fast contact lookups
CREATE INDEX idx_phone_log_number ON phone_log(phone_number);
CREATE INDEX idx_phone_log_contact ON phone_log(contact_id);

-- Function: auto-match phone number to contact
CREATE OR REPLACE FUNCTION match_phone_contact()
RETURNS TRIGGER AS $$
BEGIN
  SELECT id, name INTO NEW.contact_id, NEW.contact_name
  FROM contacts
  WHERE REPLACE(REPLACE(REPLACE(phone, '-', ''), '(', ''), ')', '') 
      = REPLACE(REPLACE(REPLACE(NEW.phone_number, '-', ''), '(', ''), ')', '')
  LIMIT 1;
  
  IF NEW.contact_id IS NOT NULL THEN
    NEW.matched := TRUE;
    -- Update last_contact_date on the contact
    UPDATE contacts SET last_contact_date = CURRENT_DATE, updated_at = NOW()
    WHERE id = NEW.contact_id;
  END IF;
  
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER auto_match_phone
  BEFORE INSERT ON phone_log
  FOR EACH ROW EXECUTE FUNCTION match_phone_contact();
```

# STEP 2: ADD API ROUTES

### /api/phone-log/route.ts
Receives webhooks from MacroDroid. No auth (MacroDroid can't send bearer tokens easily), but validate with a shared secret in the URL.

```typescript
// src/app/api/phone-log/route.ts
import { createClient } from '@supabase/supabase-js';
import { NextRequest, NextResponse } from 'next/server';

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_KEY!
);

export async function POST(req: NextRequest) {
  // Simple secret validation via query param
  const secret = req.nextUrl.searchParams.get('key');
  if (secret !== process.env.MACRODROID_SECRET) {
    return NextResponse.json({ error: 'unauthorized' }, { status: 401 });
  }

  const body = await req.json();
  // body shape: { type, phone_number, duration_seconds?, sms_body?, logged_at }

  const { data, error } = await supabase
    .from('phone_log')
    .insert({
      phone_number: body.phone_number,
      type: body.type,
      duration_seconds: body.duration_seconds || null,
      sms_body: body.sms_body || null,
      logged_at: body.logged_at || new Date().toISOString(),
    })
    .select()
    .single();

  if (error) {
    return NextResponse.json({ error: error.message }, { status: 500 });
  }

  return NextResponse.json({ 
    logged: true, 
    matched: data.matched,
    contact_name: data.contact_name 
  });
}
```

### /api/send-sms/route.ts
TCC frontend calls this → triggers MacroDroid on Tony's phone to send SMS.

```typescript
// src/app/api/send-sms/route.ts
import { createClient } from '@supabase/supabase-js';
import { NextRequest, NextResponse } from 'next/server';

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_KEY!
);

export async function POST(req: NextRequest) {
  const { phone_number, message, contact_id } = await req.json();

  // Option A: Fire MacroDroid webhook to send SMS from Tony's phone
  // MacroDroid listens on a local HTTP server or via Firebase Cloud Messaging
  // For simplicity, we use the "Webhook (URL Trigger)" in MacroDroid
  try {
    await fetch(process.env.MACRODROID_WEBHOOK_URL!, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ phone_number, message }),
    });
  } catch (e) {
    // MacroDroid webhook may not return — that's OK
  }

  // Log the outbound SMS
  await supabase.from('phone_log').insert({
    phone_number,
    contact_id,
    type: 'sms_outbound',
    sms_body: message,
    logged_at: new Date().toISOString(),
  });

  return NextResponse.json({ sent: true });
}
```

### SMS Compose Modal (new component):

```tsx
// src/components/SmsModal.tsx
function SmsModal({ contact, onClose }) {
  const [message, setMessage] = useState('');
  const [sending, setSending] = useState(false);

  async function handleSend() {
    setSending(true);
    await fetch('/api/send-sms', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        phone_number: contact.phone,
        message,
        contact_id: contact.id,
      }),
    });
    setSending(false);
    onClose();
  }

  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center z-50">
      <div className="bg-white rounded-lg p-6 w-full max-w-md">
        <h3 className="font-bold text-lg mb-2">Text {contact.name}</h3>
        <p className="text-sm text-gray-500 mb-4">{contact.phone}</p>
        <textarea
          value={message}
          onChange={(e) => setMessage(e.target.value)}
          placeholder="Type your message..."
          className="w-full border rounded p-3 h-32 mb-4"
        />
        <div className="flex gap-2 justify-end">
          <button onClick={onClose} className="px-4 py-2 text-gray-600">Cancel</button>
          <button onClick={handleSend} disabled={sending || !message}
            className="px-4 py-2 bg-blue-600 text-white rounded">
            {sending ? 'Sending...' : 'Send from my phone'}
          </button>
        </div>
      </div>
    </div>
  );
}
```

# STEP 5: MACRODROID SETUP ON TONY'S PHONE

Tony does these steps once (~10 minutes):

### Macro 1: "Log Outbound Call"
- **Trigger:** Call Ended → Outgoing Calls Only
- **Action:** HTTP Request
  - Method: POST
  - URL: `https://tonys-command-center.replit.app/api/phone-log?key=tcc-2026-phone`
  - Content Type: `application/json`
  - Body:
    ```json
    {
      "type": "call_outbound",
      "phone_number": "{call_number}",
      "duration_seconds": {call_duration_s},
      "logged_at": "{year}-{month_digit}-{dayofmonth}T{hour24}:{minute}:{second}Z"
    }
    ```

### Macro 2: "Log Inbound Call"
- **Trigger:** Call Ended → Incoming Calls Only
- **Action:** HTTP Request
  - Method: POST
  - URL: `https://tonys-command-center.replit.app/api/phone-log?key=tcc-2026-phone`
  - Content Type: `application/json`
  - Body:
    ```json
    {
      "type": "call_inbound",
      "phone_number": "{call_number}",
      "duration_seconds": {call_duration_s},
      "logged_at": "{year}-{month_digit}-{dayofmonth}T{hour24}:{minute}:{second}Z"
    }
    ```

### Macro 3: "Log Incoming SMS"
- **Trigger:** SMS Received
- **Action:** HTTP Request
  - Method: POST
  - URL: `https://tonys-command-center.replit.app/api/phone-log?key=tcc-2026-phone`
  - Content Type: `application/json`
  - Body:
    ```json
    {
      "type": "sms_inbound",
      "phone_number": "{sms_number}",
      "sms_body": "{sms_body}",
      "logged_at": "{year}-{month_digit}-{dayofmonth}T{hour24}:{minute}:{second}Z"
    }
    ```

### Macro 4: "Send SMS from TCC"
- **Trigger:** Webhook (URL Trigger) → macro name: `send-sms`
- **Action:** Send SMS
  - Number: `{webhook_payload:phone_number}`
  - Message: `{webhook_payload:message}`

### Required Android Permissions:
- Phone (call log access)
- SMS (read + send)
- Battery optimization: EXEMPT MacroDroid (Settings → Battery → MacroDroid → Unrestricted)
- Background activity: ALLOW (Settings → Apps → MacroDroid → allow background)

# CRITICAL NOTES

1. **MacroDroid MUST be exempted from battery optimization.** Android will kill it otherwise and webhooks stop firing. This is the #1 failure mode.

2. **The webhook URL must use HTTPS.** Replit deployments are HTTPS by default.

3. **The secret key in the URL is basic security.** It prevents random POST requests from logging fake calls. Not bulletproof, but sufficient for a single-user app.

4. **Call duration from MacroDroid:** The `{call_duration_s}` magic text gives seconds. If unavailable on the specific Android version, use the Call Active trigger to store a start timestamp and calculate duration on Call Ended.

5. **SMS body encoding:** MacroDroid's `{sms_body}` may contain special characters. The API route should handle JSON parsing gracefully.

6. **Contact matching depends on clean phone data.** The Supabase trigger strips dashes and parentheses, but phone numbers in the contacts table should ideally be stored in a consistent format (e.g., +19095551234).
