## Fix 1: UTC Date â†’ Pacific Timezone (Breaks check-in after 5 PM)

**Problem:** `new Date().toISOString().split("T")[0]` returns UTC. At 11 PM Pacific = tomorrow in UTC. Tony's check-in shows "not done" even though he completed it.

**Files:** Every route that does date comparison â€” `checkin.ts`, `journal.ts`, `brief.ts`, `eod.ts`, `tasks.ts`, `demos.ts`, `emails.ts`

**Fix:** Create a shared helper and use it everywhere:

```typescript
// artifacts/api-server/src/lib/dates.ts (NEW FILE)
export function todayPacific(): string {
  return new Date().toLocaleDateString("en-CA", { timeZone: "America/Los_Angeles" });
  // Returns "2026-04-04" format, always Pacific time
}
```

Replace every instance of:
```typescript
const today = new Date().toISOString().split("T")[0];
```
With:
```typescript
import { todayPacific } from "../../lib/dates";
const today = todayPacific();
```

**Effort:** 20 min

## Fix 4: Phone-Log Security (Open webhook when secret unset)

**Problem:** `checkSecret()` in `phone-log.ts` returns `true` when `MACRODROID_SECRET` env var is not set. Anyone can POST to the webhook.

**Fix:**
```typescript
// phone-log.ts â€” replace checkSecret function
function checkSecret(req: Request): boolean {
  const secret = process.env.MACRODROID_SECRET;
  if (!secret) return false; // CLOSED by default when unset
  return req.query.key === secret || req.headers.authorization === `Bearer ${secret}`;
}
```

Also move secret from query parameter to Authorization header:
```typescript
// MacroDroid should send: Authorization: Bearer YOUR_SECRET
// Instead of: ?key=YOUR_SECRET
```

## Fix 6: Add Auth Middleware (All 16 endpoints are open)

**Problem:** No authentication on any endpoint. Anyone with the URL can read Tony's emails, calendar, contacts, and post to his Slack.

**Fix:**
```typescript
// artifacts/api-server/src/middleware/auth.ts (NEW FILE)
import type { Request, Response, NextFunction } from "express";

export function requireAuth(req: Request, res: Response, next: NextFunction) {
  const token = req.headers.authorization?.replace("Bearer ", "");
  if (!token || token !== process.env.TCC_API_TOKEN) {
    res.status(401).json({ error: "Unauthorized" });
    return;
  }
  next();
}
```

Apply to all routes in `app.ts`:
```typescript
import { requireAuth } from "./middleware/auth";
app.use("/api", requireAuth);
```

Set `TCC_API_TOKEN` in Replit secrets. Frontend sends it in `lib/api.ts`:
```typescript
headers: { 
  "Content-Type": "application/json",
  "Authorization": `Bearer ${import.meta.env.VITE_TCC_TOKEN}`
}
```

## Fix 7: Add Error Boundary (White-screen crash prevention)

**Problem:** Any uncaught error in any component crashes the entire app to a white screen.

**Fix in `artifacts/tcc/src/main.tsx`:**
```typescript
import { Component, type ReactNode } from "react";

class ErrorBoundary extends Component<{ children: ReactNode }, { hasError: boolean; error?: Error }> {
  state = { hasError: false, error: undefined as Error | undefined };
  static getDerivedStateFromError(error: Error) { return { hasError: true, error }; }
  render() {
    if (this.state.hasError) {
      return (
        <div style={{ padding: 40, textAlign: "center", fontFamily: "sans-serif" }}>
          <h2>Something went wrong</h2>
          <p style={{ color: "#666" }}>{this.state.error?.message}</p>
          <button onClick={() => window.location.reload()}>Reload</button>
        </div>
      );
    }
    return this.props.children;
  }
}

// Wrap <App /> with it:
<ErrorBoundary><App /></ErrorBoundary>
```
