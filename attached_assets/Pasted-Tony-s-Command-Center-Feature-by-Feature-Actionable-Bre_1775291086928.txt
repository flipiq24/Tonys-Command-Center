CROSS-CUTTING: App.tsx Architecture
The single biggest structural issue affecting ALL features:

SharedModals and SharedHeader defined inside render body (App.tsx:233-255)

React sees these as NEW component types every render
Every parent re-render unmounts and remounts them — destroying internal state, causing flicker
Fix: Move to top-level components outside App(), pass props normally
All state in one component — 15+ useState calls

Any state change re-renders entire tree
Clock ticks every 30 seconds, re-rendering everything
Fix (short-term): Extract clock to its own component. Memoize child views with React.memo.
Fix (long-term): Extract to context providers or Zustand store
CheckinState duplicated in App.tsx:20 and CheckinGate.tsx

Fix: Move to types.ts
No Error Boundary — any component throw crashes entire app

Fix: Wrap <App /> in main.tsx with an error boundary
