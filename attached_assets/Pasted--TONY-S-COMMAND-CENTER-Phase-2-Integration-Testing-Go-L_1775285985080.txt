# PHASE 1: SMOKE TEST — Does anything work at all?

Run these 5 tests first. If ANY fail, stop and fix before continuing.

### Test 1: Does the app load?
```
Open the production URL in a browser.
EXPECTED: Check-in gate appears. No errors in console.
FAIL IF: White screen, error message, or any view other than check-in.
```

### Test 2: Does Supabase connect?
```
Submit a check-in with: bed "10:30 PM", wake "6:00 AM", Bible=yes.
EXPECTED: Data appears in Supabase checkins table within 5 seconds.
FAIL IF: No row in Supabase, or row has wrong data.
```

### Test 3: Does the Anthropic API respond?
```
Complete check-in + journal. Get to the emails step.
Click "Suggest Reply" on any email.
EXPECTED: Claude returns a drafted reply within 10 seconds.
FAIL IF: Timeout, error message, or no response.
```

### Test 4: Does MCP actually connect to Gmail?
```
From the app (or via /api/claude), send this message:
"Show me Tony's 3 most recent unread emails"
EXPECTED: Real email data from tony@flipiq.com returns.
FAIL IF: Error about Gmail MCP, authentication, or empty results when emails exist.
```

### Test 5: Does the state persist?
```
Complete check-in. Close the browser tab completely.
Reopen the URL.
EXPECTED: App skips check-in, shows journal (or wherever you left off).
FAIL IF: Check-in gate appears again, or state is lost.
```

**All 5 pass? → Move to Phase 2.**
**Any fail? → Fix the issue. Re-run all 5. Do not proceed.**
